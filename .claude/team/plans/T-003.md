# T-003 Plan - Rock Paper Scissors

## Goal
Create a self-contained `RockPaperScissors` React component that lets a player choose rock, paper, or scissors against a random computer choice, tracks scores across rounds, and can restart cleanly.

## Files to create/modify
- Create `src/games/RockPaperScissors.jsx`
- No other production files required.
- Optional follow-up only if the host app needs discovery: export or route the component from the app's game index, if such an index is added later.

## Public interface sketch

```jsx
export default function RockPaperScissors() {
  // no props required
}
```

Internal constants:

```js
const CHOICES = ["rock", "paper", "scissors"];
const WINNING_MATCHUPS = {
  rock: "scissors",
  paper: "rock",
  scissors: "paper",
};
```

Internal state:

```js
const [scores, setScores] = useState({
  player: 0,
  computer: 0,
  draws: 0,
});
const [lastRound, setLastRound] = useState(null);
```

Expected rendered surface:
- Three player buttons: `Rock`, `Paper`, `Scissors`.
- Score line or grouped counters for player wins, computer wins, and draws.
- Last-round display after the first choice: player's pick, computer's pick, and result text.
- `Restart` button that resets both `scores` and `lastRound`.

Result strings should be simple and acceptance-friendly:
- `You win`
- `Computer wins`
- `Draw`

## Sub-tasks in dependency order

1. Implement the component shell and constants.
   Recommended size: S. Recommended assignee: dev1 or dev4.
   Create `src/games/RockPaperScissors.jsx`, import `useState`, define choices and matchup constants, and default-export `RockPaperScissors`.

   Recommended size: S. Recommended assignee: dev1 or dev4.
   Add `getComputerChoice()`, `getRoundResult(playerChoice, computerChoice)`, and `handleChoice(playerChoice)`. Use functional `setScores` updates so repeated fast clicks cannot drop increments.

3. Build the JSX UI with inline styles matching the existing game pattern.
   Recommended size: S. Recommended assignee: dev1 or dev4.
   Follow `src/games/TicTacToe.jsx`: one `<section>`, minimal inline style object, accessible button labels, and `aria-live="polite"` on changing result/score text.

4. Add restart behavior and smoke-check the file.
   Recommended size: S. Recommended assignee: dev1.
   `restartGame()` should set scores to `{ player: 0, computer: 0, draws: 0 }` and `lastRound` to `null`. Re-read the final file end-to-end for valid JSX, balanced tags/braces, top-level hooks, and acceptance coverage.
